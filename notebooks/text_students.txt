Compreensão do Negócio: Nessa primeira fase o foco maior é entender o objetivo e os requisitos do projeto. A parte principal dessa etapa está em conhecer e compreender o problema que precisa ser resolvido. Conforme visto no diagrama, o entendimento geral do problema não necessariamente ficará limitado a uma única iteração sobre essa fase da implementação, no desenvolvimento do projeto é possível voltar dado que o grupo estará com uma compreensão maior do problema a ser resolvido.
Compreensão do Dados: Ocorre de maneira concorrente com a etapa anterior, a dupla seta mostra essa dinâmica entre essas fases. Essa etapa de estudo sobre as informações coletadas segue um fluxo de coleta dos dados iniciais, descrição dos dados, exploração, verificação da qualidade dos dados e verificação se a volumetria dos dados atende ao negócio em questão.
Preparação dos Dados: Etapa que prepara os dados para a modelagem. Alguns requisitos importantes para essa fase consistem em selecionar os dados, quais informações serão usadas/descartadas. Limpeza dos dados, o que pode envolver imputar ou remover valores incorretos. Construir dados, combinar informações da base para ter novos atributos que são mais coerentes para um resultado esperado. Formatação dos dados como transformação One-Hot Encoding em variáveis categóricas.
Modelagem: A fase de modelagem consiste em aplicação de técnicas e algoritmos de modelagem de dados que foram preparados na etapa anterior. O ajuste é crucial para uma garantir uma eficácia e precisão.
Avaliação: A fase de avaliação é em suma a saída da fase anterior com um aprimoramento da acurácia do resultado obtido anteriormente. São utilizadas técnicas como validação cruzada e outras métricas de desempenho para avaliar o quão bom é o modelo com os dados de teste. A partir dos resultados, pode-se ajustar ou melhorar o modelo.
Desenvolvimento: Etapa final do processo no final de uma iteração inicial. É feita a implementação do modelo em um ambiente de produção e monitoramento do seu desempenho.
